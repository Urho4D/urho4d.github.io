But would it need public testing and stability before being ready to be in an official release?

It think it would be a good idea to follow the famous Vincent Driessen’s Git workflow so we could make this available in the development branch (or feature / type branch) and the master branch would be judged stable thought it might need public testing before being sent to a tag release.

Using this standard, you could also create specific branch namespace such as CI (“ ci / target ”) too and maybe a test namespace (“ test / feature ”) to invite people to test a specific feature.

This is just a suggestion and it might not fit your habits and/or structure but I’ve seen some projects on Github following this standard such as libtomcrypt.

BTW, I also agree that a new release would be good to have soon.
