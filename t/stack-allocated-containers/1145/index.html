There is no reason why it shouldn’t be as long as you are still using pointers to urho3d objects that urho3d needs to keep track of. But for anything that the urho3d won’t need to keep track of such as temporary objects can be allocated on the stack. It should work just fine but you will run into problems that the blog post shows when you try to pass such values.
